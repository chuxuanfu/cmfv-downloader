# 脚本更新说明

## 🔄 更新内容

已将 `download_merge.sh` 更新为**可靠版本**，使用和 `apple-video-downloader` 相同的成功方法。

### ✨ 主要改进

#### 1. **串行下载（取代并发）**

**之前：** 并发下载10个分段
```bash
curl --parallel --parallel-max 10 ...
```

**现在：** 一个一个下载，确保每个都成功
```bash
for i in $(seq $START_SEGMENT $END_SEGMENT); do
    download_with_validation ...
done
```

#### 2. **文件大小验证**

**新增功能：** 每个文件下载后检查大小

```bash
MIN_VIDEO_SIZE=100000    # 视频分段必须 > 100KB
MIN_AUDIO_SIZE=10000     # 音频分段必须 > 10KB
MIN_VTT_SIZE=100         # 字幕分段必须 > 100B
```

如果文件太小（说明下载失败或损坏），自动删除并重试。

#### 3. **自动重试机制**

**最多重试5次：**

```bash
download_with_validation() {
    local max_retries=5
    for ((retry=1; retry<=max_retries; retry++)); do
        # 下载
        # 验证大小
        # 成功返回，失败重试
    done
}
```

每次重试间隔 1 秒，避免服务器限流。

#### 5. **智能断点续传**

**自动检测已下载的有效分段：**

```bash
# 检查文件是否存在且大小足够
if [[ -f "$OUTPUT_FILE" ]]; then
    SIZE=$(stat -f%z "$OUTPUT_FILE")
    if [[ $SIZE -ge $MIN_VIDEO_SIZE ]]; then
        SKIPPED=$((SKIPPED + 1))  # 跳过
        continue
    fi
fi
```

中断后重新运行，只下载缺失或损坏的分段。

#### 6. **拼接时过滤损坏文件**

```bash
for i in $(seq $START_SEGMENT $END_SEGMENT); do
    if [[ -f "$F" ]]; then
        SIZE=$(stat -f%z "$F")
        if [[ $SIZE -ge $MIN_VIDEO_SIZE ]]; then
            cat "$F" >> "$MERGED_VIDEO"  # 只拼接有效文件
        fi
    fi
done
```

确保不会把损坏的分段加入最终视频。

#### 7. **完整性验证**

**下载后自动验证：**

```bash
V_DUR=$(ffprobe ... -show_entries stream=duration ...)
TOTAL_DUR=$(ffprobe ... -show_entries format=duration ...)

DIFF=$(echo "$TOTAL_DUR - $V_DUR" | bc | tr -d '-')

if (( DIFF < 5 )); then
    echo "✅✅✅ 成功！视频完整，可以正常播放！"
else
    echo "⚠️  警告：视频流时长和总时长差异"
fi
```

确保视频和音频时长匹配，没有黑屏问题。

## 🐛 已解决的问题

### 问题 1: 部分分段只有1字节

**原因：** 服务器返回错误但 curl 没有报错

**解决：** 检查文件大小，< 100KB 认为失败

### 问题 2: 并发下载导致限流

**原因：** 同时请求太多被服务器拒绝

**解决：** 改为串行下载，每个请求间隔 0.1 秒

### 问题 3: 视频拼接后有黑屏

**原因：** 损坏的分段被拼接进视频

**解决：** 拼接前过滤，只拼接有效分段

### 问题 4: 下载失败无法重试

**原因：** 旧版本没有重试机制

**解决：** 每个分段最多重试 5 次
